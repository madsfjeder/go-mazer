# go-mazer - a maze generator + solver written entirely in Go

<img width="800" height="800" alt="maze1" src="https://github.com/user-attachments/assets/1e028e90-0443-4759-9dfe-9a6489a1c00a" />


A simple learning project written to learn Go, and brush up on searching algorithms. 
All graphics drawn using [raylib-go](https://github.com/gen2brain/raylib-go)

Execution is simple: Run `go run main.go` and you'll immediately see a maze being randomly generated and solved. 

Available algorithms:

- DFS (Depth first search) - guaranteed to find the a path (but can be very ineffecient)
- BFS (Breadth first search) - guaranteed to fint the shortest path (but can be very ineffecient)
- GFS (Greedy first search) - fast but not guaranteed to find a path
- A* - guaranteed to find the fastest path (more optimal in terms of search space, but can be slow computationally)

Available levels:

- Randomly generated maze. This is generated by DFS algorithm paired with a homegrown "splitter" concept, where the path will "split" randomly at intervals, creating branching paths.
More splits = more complex maze
- An empty test level. This is great for seeing how the different algos behave on a visual level (also where BFS really shows its ineffeciency, and GFS shines)
